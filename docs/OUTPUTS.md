# SimToPC Outputs

## generate

Running:

    simtopc generate config.yml

creates a folder named as the user-defined `mesh_density` parameter
(e.g. `MESH_DENSITY/`) in the repository root.

Inside this folder, one subfolder is created per parameter set defined
by the user:

    MESH_DENSITY/
      test_case_1/
      test_case_2/
      test_case_3/

Each `test_case_i/` folder corresponds to one OpenFOAM simulation case
generated and executed by SimToPC.



## measure

Running:

    simtopc measure config.yml

post-processes each OpenFOAM simulation case generated by the
`generate` command (e.g. `MESH_DENSITY/test_case_i/`) to extract
melt-pool geometrical quantities along the entire scan track.

The measurement workflow proceeds in two main stages:

1. **Track continuity check**  
   Using the known regular mesh spacing, the expected cross-sections
   along the scan track are enumerated.
   If any expected cross-section contains no material cells, the track
   is classified as non-continuous.
   In this case, only a boolean continuity flag is stored and no
   further geometrical analysis is performed for that simulation case.

2. **Per-section geometrical analysis (continuous tracks only)**  
   For continuous tracks, melt-pool geometry is evaluated at each
   cross-section along the track.
   The following quantities are computed:

   - melt pool width (W)
   - melt pool height (H)
   - melt pool depth (D)
   - porosity

The computed quantities are written to disk as text or CSV files
associated with each simulation case.




## surrogate

Running:

    simtopc surrogate config.yml

uses the measurements computed by the `measure` command to build
a surrogate model based on a neural network.

The surrogate model is trained using the melt pool quantities
(W, D, H, porosity) obtained from the simulation cases.

The command produces:

- a dataset used for training the surrogate model
- a trained neural network model saved to disk
- files containing training metrics and/or predictions
